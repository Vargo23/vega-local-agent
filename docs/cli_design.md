# CLI Design для VEGA

## Цель

CLI-оформление нужно не для понтов, а для читаемости.

Пользователь должен сразу видеть:

- агент запущен;
- какая модель активна;
- какой режим работы включен;
- интернет включен или выключен;
- что агент делает сейчас;
- где ошибка;
- какой блок готов;
- что ушло на переработку.

## Стартовый баннер

```text
╔════════════════════════════════════╗
║              VEGA v0.1             ║
║        Local Project Agent         ║
╚════════════════════════════════════╝

Model: qwen2.5-coder
Mode: coding-agent
Internet: OFF
Project: current

Status: Ready
> Введите задачу для VEGA.
```

## Статусы

Использовать короткие теги:

```text
[USER]      задача пользователя
[VEGA]      ответ агента
[PLAN]      план действий
[CODE]      код или файл
[TOOL]      действие инструмента
[REVIEW]    проверка координатора
[DONE]      блок готов
[REWORK]    нужна переработка
[WARNING]   ограничение или риск
[ERROR]     ошибка
```

## Пример нормального вывода

```text
[USER] Добавь обработку ошибок в модуль загрузки файлов.

[VEGA] Задача понята: нужно добавить безопасную обработку ошибок при чтении файлов.

[PLAN]
1. Найти точку загрузки файла.
2. Добавить проверку существования файла.
3. Добавить обработку ошибок чтения.
4. Проверить, что ошибка возвращается понятно.

[REVIEW] План соответствует задаче. Можно выполнять Block 1.
```

## Пример статуса готового блока

```text
[DONE] Block 1 готов: добавлена проверка существования файла.
[REVIEW] Блок принят. Можно переходить к Block 2.
```

## Пример переработки

```text
[REWORK] Block 2 не принят.
Причина: ошибка обрабатывается, но пользователь не получает понятное сообщение.
Действие: переписать обработчик ошибки и повторить review.
```

## Цвета

Цвета можно добавить позже через Python-библиотеки `rich` или `colorama`.

Рекомендуемая логика:

- `[DONE]` — зеленый;
- `[ERROR]` — красный;
- `[WARNING]` — желтый;
- `[REVIEW]` — синий или фиолетовый;
- `[USER]` — обычный;
- `[VEGA]` — обычный или cyan.

Цвета не должны ломать читаемость в обычном терминале.
