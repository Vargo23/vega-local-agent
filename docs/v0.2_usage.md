# VEGA v0.2 Usage

## Start VEGA

Open a terminal in the project root and run:

```powershell
python .\scriptsega.py
```

You should see:

```text
VEGA v0.2
Local Project Agent
Model: vega-core
Internet: OFF
Status: Ready
```

You can also start VEGA with:

```powershell
.un_vega.ps1
```

or:

```bat
run_vega.bat
```

## Create the internal model

VEGA CLI uses the internal Ollama model `vega-core`. Create it from the project root:

```powershell
ollama create vega-core -f .\ollama\Modelfile
```

## Check that the model exists

Run:

```powershell
ollama list
```

Look for `vega-core` or `vega-core:latest` in the output.

## Send messages

Type a message at the prompt:

```text
VEGA> help me design a small Python module
```

The CLI sends the active session message list to:

```text
http://localhost:11434/api/chat
```

The current session keeps context in memory until you exit VEGA.

## Why does `ollama run vega` not show the banner?

`ollama run` starts only an Ollama model. The banner, commands, logs, and session management live in the Python CLI shell. Therefore the main launch for VEGA v0.2 is:

```powershell
python .\scriptsega.py
```

If you run a model directly with `ollama run`, you are in raw Ollama mode. Raw mode has no VEGA CLI banner, no `/status`, no `/log`, and no CLI-managed session memory.

## Exit VEGA

Use either command:

```text
/exit
```

or:

```text
/bye
```

`Ctrl+C` also closes the session.

## Logs

Logs are stored in:

```text
logs\sessions
```

Each session creates a separate `.txt` file. The log contains the start date, project path, model name, user messages, and VEGA responses.

## Available commands

- `/help` - show available commands
- `/status` - show CLI version, model, project path, internet mode, log path, and context status
- `/model` - show the internal Ollama model and explain the correct launch path
- `/project` - show the project path
- `/clear` - clear the terminal screen
- `/log` - show the current session log file
- `/exit` - exit VEGA
- `/bye` - exit VEGA

## If the Ollama API is unavailable

Make sure Ollama is running. VEGA will print:

```text
Ollama API is unavailable.
Check that Ollama is running.
Then try:
ollama list
```

## If the VEGA Core model is not found

Create the model from the project root:

```powershell
ollama create vega-core -f .\ollama\Modelfile
```

VEGA v0.2 does not create, rename, or remove local Ollama models automatically.
