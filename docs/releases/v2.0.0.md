# VEGA v2.0.0 Release Notes

## Agent Orchestration Foundation

VEGA v2.0.0 provides the architectural foundation for controlled agent orchestration. It is not a fully autonomous agent.

## Added

* Deterministic intent and slash-command routers.
* A shared execution context for each VEGA session.
* A process-local confirmation manager with one pending action.
* A top-level orchestrator that returns structured routing results.
* An isolated local Ollama HTTP client.
* Unit and integration coverage for the orchestration layer.

## Changed

* `scripts/vega.py` is now a thin CLI entrypoint.
* Interactive runtime behavior lives in `core/agent_runtime.py`.
* Chat history, Agent Mode, and Project Memory context are assembled by the runtime after deterministic routing.

## Security

* Ordinary chat cannot be promoted to a slash command by the model.
* Model-driven automatic tool execution is not enabled.
* Existing workspace, terminal, internet, patch, confirmation, and Git policies remain authoritative.

## Scope

This release establishes routing, shared session state, runtime isolation, and Ollama client boundaries. Structured command execution and controlled tool orchestration are planned for v2.1.0.
