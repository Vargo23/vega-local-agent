# VEGA v2.1.0 Release Notes

## Overview

VEGA v2.1.0 adds structured slash-command execution and controlled orchestration for selected read-only tools. It does not make VEGA a fully autonomous agent.

## Added

* `CommandExecutor` with typed requests, results, and statuses.
* `ToolExecutor` with signature validation and controlled failure results.
* Runtime and command-handler integration tests.

## Changed

* Routed slash commands execute through `CommandExecutor`.
* `/docs` uses a dedicated compatibility adapter.
* `/file`, `/git`, and `/tools list` use one session-level `ToolExecutor`.
* Command execution errors are logged as `COMMAND_ERROR`.

## Security

* Tool names are selected by fixed handler mappings, not user or model output.
* Users cannot run an arbitrary registered tool.
* The model and `AgentOrchestrator` do not receive `ToolExecutor`.
* Automatic model-driven tool calling and autonomous execution loops remain disabled.
* Existing write, terminal, test, internet, web, patch, and confirmation policies remain authoritative.

## Scope

The controlled tool path is limited to the read-only `/file`, `/git`, and `/tools list` commands. Coding workflows and broader controlled orchestration are deferred to v2.2.0.
